Protected processEvent

  • processEvent(event: SentryEvent, send: function, hint?: SentryEventHint, scope?: Scope): Promise<SentryResponse>

  • Inherited from BaseClient.processEvent

    • Defined in core/src/baseclient.ts:340

    Processes an event (either error or message) and sends it to Sentry.

    This also adds breadcrumbs and context information to the event. However, platform specific meta data (such as the User's IP address) must be added by the SDK implementor.

    The returned event status offers clues to whether the event was sent to Sentry and accepted there. If the {@link Options.shouldSend} hook returns false, the status will be {@link SendStatus.Skipped}. If the rate limit was exceeded, the status will be {@link SendStatus.RateLimit}.

    Parameters

    • event: SentryEvent

      The event to send to Sentry.

    • send: function

      A function to actually send the event.

        • (finalEvent: SentryEvent): Promise<SentryResponse>

        • Parameters

          • finalEvent: SentryEvent

          Returns Promise<SentryResponse>

    • Optional hint: SentryEventHint

      May contain additional informartion about the original exception.

    • Optional scope: Scope

      A scope containing event metadata.

    Returns Promise<SentryResponse>

    A Promise that resolves with the event status.
