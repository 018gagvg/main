debug: object

This is only if you want to debug in prod. writeAsBreadcrumbs: Instead of having console.log statements we log messages to breadcrumbs so you can investigate whats happening in production with your users to figure why things might not appear the way you expect them to.

idleTimeout

idleTimeout: number
  • Defined in apm/src/integrations/tracing.ts:55

The time to wait in ms until the transaction will be finished. The transaction will use the end timestamp of the last finished span as the endtime for the transaction. Time is in ms.

Default: 500

markBackgroundTransactions

markBackgroundTransactions: boolean
  • Defined in apm/src/integrations/tracing.ts:88

Flag Transactions where tabs moved to background with "cancelled". Browser background tab timing is not suited towards doing precise measurements of operations. Background transaction can mess up your statistics in non deterministic ways that's why we by default recommend leaving this opition enabled.

Default: true

maxTransactionDuration

maxTransactionDuration: number
  • Defined in apm/src/integrations/tracing.ts:79

The maximum duration of a transaction before it will be marked as "deadline_exceeded". If you never want to mark a transaction set it to 0. Time is in seconds.

Default: 600
